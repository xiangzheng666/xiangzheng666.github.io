# 操作系统

- 操作系统流程
  - 通过二进制编辑器编辑二进制源代码文件(img)
  - ![1659919746907](./img/1659919746907.png)
  - 将二进制文件写入软盘，设置软盘启动
  
- 使用汇编语言编写源代码
  - 编译器nask
    - **nask.exe helloos.nas helloos.img**
    - **DB指令是“define byte”的缩写，也就是往文件 里直接写入1个字节的指令**
    - **RESB指令是“reserve byte”的略写，如果想要从 现在的地址开始空出10个字节来，就可以写成 RESB 10**
    - **DW指令和DD指令，它们分别是“define word”和“ddefine double-word”的缩写：2字节 4字节**
  - ![1659920089073](./img/1659920089073.png)
  
- 使用引导区ipl
  - ![1668047032668](./img/1668047032668.png)
  - 考虑到以后的开发，我们不要一下子就用nask来做整个磁盘映像，而是先只用它来制作512字节的启动区，剩下的部分我们用磁盘映像管理工具来做， 这样以后用起来就方便了。
  - 启动区只需要最初的512字节
  - 调用磁盘BIOS 的中断程序，进行读盘操作，加载程序
  - 实模式--》保护模式
  
- 使用makefile文件管理命令

  - 自动检查文件更新状态

  - ```
    \#文件生成规则 
    
    ipl.bin : ipl.nas Makefile 
    ../z_tools/nask.exe ipl.nas ipl.bin ipl.lst 
    
    helloos.img : ipl.bin Makefile 
    ../z_tools/edimg.exe 
    
    imgin:../z_tools/fdimg0at.tek wbinimg src:ipl.bin len:512 from:0 to:0 
    
    imgout:helloos.img
    #命令
    img :
    ../z_tools/make.exe -r helloos.img
    
    asm :
    ../z_tools/make.exe -r ipl.bin
    
    run :
    ../z_tools/make.exe img 
    copy helloos.img ..\z_tools\qemu\fdimage0.bin
    ../z_tools/make.exe -C ../z_tools/qemu
    
    install :
    ../z_tools/make.exe img
    ../z_tools/imgtol.com w a: helloos.img
    ```
